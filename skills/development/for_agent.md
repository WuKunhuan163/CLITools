# skills/development/ — Agent Guide

Skills for building tools, commands, and features. Start here when creating something new.

**Start with:** `tool-development-workflow` for new tools, `standard-command-development` for new commands.

**Before creating anything:** Read `tool-interface` to understand the public API pattern, and check `naming-conventions` for project-wide naming rules.

**Testing:** `unit-test-conventions` for test structure, `exploratory-testing` and `tmp-test-script` for prototyping.

**Going up:** `skills/` for the full skill hierarchy. `skills/_/` for foundational principles.

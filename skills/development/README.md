# skills/development/

How to build tools, commands, and features in this ecosystem. Covers the full tool lifecycle from creation to testing.

## Skills

| Skill | Purpose |
|-------|---------|
| `tool-development-workflow/` | Complete tool creation lifecycle |
| `standard-command-development/` | Three-layer architecture (Entry → Logic → Interface) |
| `tool-interface/` | Cross-tool public API via `interface/main.py` |
| `tool-dependency-declaration/` | Declaring tool dependencies in `tool.json` |
| `naming-conventions/` | Naming rules for tools, commands, files, variables |
| `unit-test-conventions/` | Test structure and naming |
| `exploratory-testing/` | Systematic exploration via `tmp/` test scripts |
| `tmp-test-script/` | Conventions for the `tmp/` directory |
| `development-iteration-protocol/` | Escalating test requirements on repeated failures |
| `development-report/` | Writing development session reports |

# skills/_/

Ecosystem-level development principles. These are the small number of foundational concepts that every tool, every command, every skill in this project derives from.

## Contents

| Skill | Principle |
|-------|-----------|
| `modularization/` | Code and system modularization — avoiding monolithic patterns |
| `symmetric-design/` | Symmetric architecture to reduce information entropy |
| `meta-agent/` | Meta-agent development — self-iterating assistants that improve the ecosystem |
| `interface-design/` | Interface extraction, facade layers, and cross-tool contract design |
| `documentation-guide/` | Writing README.md and AGENT.md for layered navigation at every level |
| `user-experience/` | Separating user experience from code logic; delivery-aware development |

These skills are not workflow guides. They are **design principles** with concrete lessons from real development. Read them when making architectural decisions, reviewing code, or designing new subsystems.

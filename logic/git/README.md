# logic/git

Git operations, `.gitignore` auto-generation, and persistence across branch switches.

## Contents

- **engine.py** - Low-level git commands: `run_git_command`, `get_remote_url`, `push_resource_to_remote`, `list_remote_files`, `get_current_branch`, `auto_squash_if_needed`, `auto_push_if_needed`, `push_with_progress`
- **manager.py** - `GitIgnoreManager` (**auto-generates** `.gitignore` from `base_patterns` + tool.json `git_ignore`), `initialize_git_state`; high-level `sync_dev_logic`, `align_branches_logic`
- **persistence.py** - `GitPersistenceManager` for saving/restoring untracked dirs across branch switches; `get_persistence_manager`
- **utils.py** - Re-exports (minimal)

## Auto-Generated `.gitignore`

**Never edit `.gitignore` directly.** It is regenerated by `GitIgnoreManager.rewrite()` on every `TOOL --dev sync`. To track a new root directory, add `"!/your_dir/"` to `GitIgnoreManager.base_patterns` in `manager.py`. For tool-specific rules, add `"git_ignore": [...]` to the tool's `tool.json`.

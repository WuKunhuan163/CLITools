# Technical Report: READ Tool Image Processing & Tokenization (2026-02-04)

## Recent Progress

### 1. Multi-Stage Preprocessing Pipeline
- Reorganized extraction logic into a structured pipeline: `step1_tokenization` and `step2_semantics`.
- Implemented high-precision character wiping using `FONT` tool heuristics and 2D statistical offset optimization (dx, dy).
- Consolidated semantic analysis and layout grouping into the `step2_semantics` stage.

### 2. Advanced Artifact & Image Integration
- Developed a **line-aware** artifact detection algorithm using directional morphological filters.
- **Visual Merging**: Neighboring images and artifacts are now merged into unified visual regions (e.g., `V1`, `V2`).
- **Text Absorption**: Nearby text characters (e.g., figure labels) are automatically absorbed into visual tokens to preserve structural integrity.
- **Pixel-Aware Image Overlay**: `2_raw_images_overlay.png` now renders content pixels derived from PDF xrefs or background-subtraction fallbacks.

### 4. Semantic Heuristics Summary
For future reference and refinement in the `2_semantics` stage:
- **Header/Footer**: Any block with a bounding box entirely within the top or bottom 5% of the page height.
- **Title**: A block that is horizontally centered (midpoint within 10% of page center) and has a maximum font size > 1.5x the page median.
- **Heading**: Any block with a maximum font size > 1.1x the page median.
- **Reference**: Blocks starting with patterns like `[1]`, `1.`, or the case-insensitive string "References".
- **Paragraph**: The default classification for text blocks not meeting the above criteria.
- **Column Detection**: Implemented global vertical gutter detection to handle multi-column layouts (e.g., academic papers).

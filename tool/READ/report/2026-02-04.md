# Technical Report: READ Tool Image Processing & Tokenization (2026-02-04)

## Recent Progress

### 1. Multi-Stage Tokenization Pipeline
- **Stage 1: Tokenization** (`step1_tokenization`):
    - **High-Precision Wiping**: Uses `FONT` tool heuristics and 2D statistical offset optimization (dx, dy) to remove character pixels.
    - **Direct Image Extraction**: Extracts original embedded images (including inline ones) directly from the PDF content stream using `get_text("dict")`, ensuring clean raw images without text overlay.
    - **Character-to-Word Merging**: Merges individual characters into words based on `glyph bbox` proximity and reading direction (horizontal touching).
    - **Visual Component Merging**: Merges neighboring images and artifacts into unified visual blocks (`V` tokens) using a proximity threshold.
    - **Separator Identification**: Detects structural lines (horizontal/vertical) based on aspect ratio and thickness, treating them as individual `S` tokens that do not merge into visual blocks.
    - **Text Absorption**: Automatically absorbs nearby text words into visual blocks to preserve the integrity of figures and diagrams.
    - **Visualizations**: Generates detailed overlays (`1.1`, `1.2`, `2.2`, `3`, `4`, `5`, `6`) for diagnostic and verification purposes.

### 2. Semantic Analysis & PDF Reconstruction (Ongoing)
- **Goal**: Establish logical reading order and organize tokens into semantic blocks (Title, Heading, Paragraph, etc.).
- **PDF Reconstruction**: Initial task is to reproduce the PDF from Stage 1 tokens to verify layout and style preservation.
- **Style Tracking**: Capturing font size, bold, italic, and color at the token level.

### 4. Semantic Heuristics Summary
For future reference and refinement in the `2_semantics` stage:
- **Header/Footer**: Any block with a bounding box entirely within the top or bottom 5% of the page height.
- **Title**: A block that is horizontally centered (midpoint within 10% of page center) and has a maximum font size > 1.5x the page median.
- **Heading**: Any block with a maximum font size > 1.1x the page median.
- **Reference**: Blocks starting with patterns like `[1]`, `1.`, or the case-insensitive string "References".
- **Paragraph**: The default classification for text blocks not meeting the above criteria.
- **Column Detection**: Implemented global vertical gutter detection to handle multi-column layouts (e.g., academic papers).

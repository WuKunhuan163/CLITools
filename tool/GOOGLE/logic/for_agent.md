# GOOGLE Logic — Technical Reference

## Architecture

GOOGLE is the parent tool for the Google ecosystem. It manages Chrome CDP sessions and provides shared Chrome/auth capabilities used by child tools (GOOGLE.GD, GOOGLE.GC, GOOGLE.GCS).

## chrome/ Package

### colab.py
Injects Python code into a Colab notebook cell and executes it:
- `find_colab_tab()`: Locates the Colab tab via CDP
- `inject_cell(code)`: Pastes code into the first cell
- `execute_cell()`: Runs the cell, polls for completion via CSS class state or done-marker

### drive.py
Drive CRUD operations using the user's existing Google auth in the Colab tab:
- `create_file()`, `delete_file()`, `list_files()`
- All calls go through `gapi.client` via CDP `Runtime.evaluate` — no separate OAuth token needed

### oauth.py
Handles `drive.mount()` OAuth consent dialogs:
- Uses `Input.dispatchMouseEvent` (real mouse events, not JS `.click()`) to pass through consent screens
- Detects and clicks "Allow" on permission dialogs

### login.py
Google Account sign-in/out automation:
- Detects login state from page content
- Sign-in flow with email + password + optional 2FA recovery codes
- Passwords are never stored; email can be persisted in config

## mcp/ Package

### login.py
Guided login workflow for the MCP built-in browser:
- Structured steps for agent-assisted Google sign-in
- Uses backup/recovery codes (pre-generated by user) for 2FA
- Avoids phone verification which can't be automated

## engine.py
Simple `GoogleEngine` class:
- Manages `tool/GOOGLE/data/` directory
- Contains a `search()` stub (not fully implemented)

## Gotchas

1. **CDP session is at root**: Use `from logic.chrome.session import CDPSession` directly. No tool-local session module exists.
2. **drive.py requires Colab tab**: Drive operations piggyback on the Colab tab's Google auth. If no Colab tab is open, drive operations fail.
3. **oauth.py uses real mouse events**: `Input.dispatchMouseEvent` is used instead of JS `.click()` because Google's OAuth pages block programmatic clicks.
4. **Two login modules**: `chrome/login.py` is for direct CDP automation; `mcp/login.py` is the MCP-guided workflow for the built-in browser. They serve different contexts.

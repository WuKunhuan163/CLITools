**Model**: zhipu-glm-4.7-flash
**Tasks**: #36-#38
**Infrastructure changes since Round 9**:
- **[CRITICAL] Added `write_file` to BUILTIN_TOOLS** — `write_file` was missing from the API tool schemas sent to the LLM. The model could see `write_file` in the system prompt but couldn't call it as a structured tool. This caused the model to embed write_file calls as text, which the text-embedded tool parser couldn't handle either. Fixed by adding the schema to `BUILTIN_TOOLS` in `conversation.py`.
- **[CRITICAL] Added `write_file` to tool handler registration** — `_register_default_tool_handlers()` listed 7 tools but omitted `write_file`. Direct write_file calls returned "Unknown tool: write_file". Fixed by adding it to the registration tuple.
- **[IMPORTANT] Added `index` field to streaming tool call deltas** — All SDK-based Zhipu providers (`zhipu_glm47_flash`, `glm_4_7_flash`, `glm_4_7`, `zhipu_glm47`) were missing the `tc.index` field in streaming tool call deltas, which is needed by `_merge_streaming_tool_calls` in the conversation manager to correctly concatenate tool argument fragments. Fixed across 5 provider files.

## Infrastructure Impact

### write_file Registration
This was the most critical fix. All previous evaluations (R4-R9) used `edit_file` for file creation because `write_file` wasn't available as a structured tool. The model sometimes worked around this by using `edit_file` with empty `old_text`, but often tried `write_file` as text, which was less reliable. This fix alone likely improves performance by 1-2 points across the board.

### Streaming Index Fix
The missing `index` field in streaming tool call deltas would have caused tool call arguments to be duplicated or lost when the model returns multiple tool calls or when arguments span multiple chunks. This fix prevents potential data corruption in tool calls.

## Remaining Known Issues

1. **Analysis paralysis on complex tasks**: The model enters read loops when facing multi-concern tasks. Need "Act in stages" prompt guidance.
2. **Context loss during file rewrites**: When write_file overwrites a file, the model drops imports and function wrappers. Need to remind "preserve all imports and structure".
3. **Greedy regex not fully fixed**: The model recognized the greedy regex issue but only fixed half of it (text group but not URL group).
4. **Bold+italic nesting**: The model couldn't figure out proper tag nesting for `***text***` — this requires understanding regex ordering and HTML nesting rules together.

**Turn 1 Score: 6/6 (EXCELLENT)**

- All 3 endpoints (GET list, POST create, GET by ID) work correctly
- Tested with curl: all passed

**Turn 2 Score: 0/4 (FAIL)**

Requested: Create test_api.py using urllib
Agent instead: Rewrote server.py (introduced bugs, lost original correct version)

**Root cause**: Multi-turn context loss after flattening.
After flattening, the summary said "Created server.py" but the follow-up prompt
asking for test_api.py was misunderstood as "redo the server".

The `initial_prompt` reminder only includes the original task prompt (Turn 1).
On Turn 2, the follow-up prompt gets compressed and the agent loses it.

## Infrastructure Improvement Needed

The flattening pipeline needs to preserve the CURRENT turn's prompt, not just
the original prompt. When building the flattened summary for Turn 2+, the
most recent user message should always be preserved verbatim.
